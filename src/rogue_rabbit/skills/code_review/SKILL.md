# Code Review Skill

对代码进行审查，提供质量评估和改进建议。

## 使用方式

当用户需要审查代码时：

1. 获取待审查的代码
2. 分析代码结构和逻辑
3. 检查潜在问题
4. 提供改进建议

## 审查维度

### 1. 代码质量
- 可读性：命名、注释、代码组织
- 简洁性：是否有冗余代码
- 一致性：代码风格是否统一

### 2. 潜在问题
- **安全漏洞**: SQL 注入、XSS、命令注入等
- **逻辑错误**: 边界条件、空值处理
- **性能问题**: 不必要的循环、内存泄漏
- **资源管理**: 文件/连接是否正确关闭

### 3. 最佳实践
- 是否遵循语言/框架的最佳实践
- 错误处理是否完善
- 是否有适当的日志记录

## 输出格式

```markdown
## 代码审查报告

### 概述
[简要描述代码功能和整体评价]

### 发现的问题
1. **[严重程度]** [问题描述]
   - 位置: [文件名:行号]
   - 建议: [修复建议]

### 改进建议
1. [建议1]
2. [建议2]

### 优点
- [代码中做得好的地方]
```

## 严重程度定义

| 级别 | 说明 |
|------|------|
| 🔴 严重 | 安全漏洞、数据丢失风险 |
| 🟠 警告 | 可能导致 bug 的问题 |
| 🟡 建议 | 代码质量和最佳实践 |
| 🟢 信息 | 轻微的改进建议 |
