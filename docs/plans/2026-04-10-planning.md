# Planning 设计方案 (v0.12.0)

**日期**: 2026-04-10
**状态**: 计划中
**前置版本**: v0.11.0 (Checkpoint & Restore)

---

## 背景

通过研究 Claude Code Plan Mode、LangGraph Plan-and-Execute 和 REWOO 等主流规划模式，提炼核心设计原理，在 RogueRabbit 中实现最小化 Planning 模块，学习"任务分解、计划生成与动态调整"的设计思想。

## 原理研究

### Claude Code Plan Mode

| 维度 | 机制 |
|------|------|
| 计划载体 | Markdown 文件（`.claude/plans/`） |
| 激活方式 | Shift+Tab 或 agent 自主进入 |
| 工具限制 | 通过 prompt 强化，不做实际修改 |
| 四阶段结构 | 理解 → 设计 → 审查 → 定稿 |

**核心原理**: 结构化 prompt + 文件载体 + 用户审批。计划本身是可读、可编辑、可持久化的文本。

### LangGraph Plan-and-Execute

| 维度 | 机制 |
|------|------|
| 状态图 | Planner → Executor → Replanner → ... → Finish |
| 计划表示 | RemainingSteps 列表 |
| 重规划 | 每步执行后评估是否需要调整 |
| 模型策略 | 规划用强模型（低温度），执行可用弱模型 |

**核心原理**: 显式状态图 + 可重规划循环。Planner 和 Executor 可以使用不同的 LLM 配置。

### REWOO (Reasoning Without Observation)

| 维度 | 机制 |
|------|------|
| 架构 | Planner → Worker → Solver |
| 关键特点 | 规划时不看工具输出，节省 token |
| 适用场景 | 已知工具链的结构化任务 |

**核心原理**: 解耦推理与观察。Planner 在不看执行结果的情况下生成完整计划。

### 提炼的设计原则

1. **先规划后执行** — 复杂任务先生成完整计划，再逐步执行
2. **动态调整** — 执行中可根据观察结果触发重规划
3. **混合模式** — 顶层 Plan-and-Execute，底层步骤可用 ReAct
4. **计划可持久化** — 结合 v0.11 Checkpoint 保存和恢复计划状态

### 规划模式图谱

```
[完全预先规划]                          [完全反应式]
REWOO  ←→  Plan-and-Execute  ←→  ReAct(已有)  ←→  Reflexion
           (带 replanning)
```

本项目重点学习 Plan-and-Execute + Replanning 模式。

## 最小实现设计

### 数据模型 (contracts/plan.py)

```
StepStatus = PENDING | IN_PROGRESS | DONE | FAILED | SKIPPED

PlanStep
├── id: str                    # 步骤标识
├── description: str           # 步骤描述
├── status: StepStatus         # 步骤状态
├── result: str | None         # 执行结果
└── observation: str | None    # 执行观察

Plan
├── plan_id: str               # 计划标识
├── goal: str                  # 目标描述
├── steps: list[PlanStep]      # 步骤列表
├── status: PlanStatus         # 计划状态 (DRAFT|ACTIVE|COMPLETED|FAILED)
├── context: str               # 规划上下文（来自 Memory/Session）
└── created_at: datetime
```

### 协议定义 (contracts/plan.py)

```
Planner (Protocol)
├── create_plan(goal, context) -> Plan
└── replan(plan, observation) -> Plan

PlanExecutor (Protocol)
├── execute_step(step) -> str
└── should_replan(plan, step_result) -> bool
```

### 管理器 (core/plan_manager.py)

```
PlanManager
├── create(goal, context?) -> Plan          # LLM 驱动生成计划
├── execute(plan) -> Plan                   # 按步骤执行
├── replan(plan, observation) -> Plan       # 动态重规划
├── get_status(plan_id) -> Plan             # 查询状态
└── to_checkpoint(plan) -> Checkpoint       # 结合 v0.11 持久化
```

### 核心循环

```
用户目标 → Planner.create_plan() → Plan
    ↓
循环:
    step = Plan.next_pending_step()
    result = PlanExecutor.execute_step(step)     # 可用 ReAct
    if should_replan(plan, result):
        PlanManager.replan(plan, result)          # 动态调整
    ↓
Plan 完成 → 结果汇总
```
